Projet restaurant

Projet d’étude consistant à mettre en place le site vitrine d’un restaurant avec un back-office. Les menus sont mis en place à l’aide d’une base de données.

Projet restaurant
  • Figma
  • HTML
  • CSS
  • Bootstrap
  • JavaScript
  • PHP
  • MySQL
  • Git
  • Github

Projet d’étude consistant à mettre en place le site vitrine d’un restaurant avec un back-office. Les menus sont mis en place à l’aide d’une base de données. Le site contient également un formulaire de contact avec envoie de mail. J'ai travaillé en binôme sur ce projet.

J’ai effectué un benchmark sur plusieurs enseignes de fast-food pour conceptualiser le zoning, le wireframe et la maquette du site sur Figma. J’ai ensuite commencé à intégrer le site en créant plusieurs pages pour chaque catégorie de plats, une page d’accueil, une page pour le formulaire et 3 pages pour le back-office, une qui affiche le contenu de la base de donnée, une pour ajouter un élément et une pour modifier un élément sélectionné. Pour l’intégration CSS de ce projet, j’ai opté pour le framework Boostrap.

Le header est composé d’un carrousel représentant le logo sur une bannière. La nav permet de se déplacer à travers toutes les pages du site. Le contenu de la page se trouve dans des balises sections qui se trouvent elles-mêmes dans une balise main. Le footer est constitué d’un plan du site ainsi que de boutons réseaux sociaux fonctionnels.

J’ai réaliser un MCD avec MySQL Workbench pour créer les tables et ainsi faire la cardinalité entre elles. Ensuite j’ai réalisé les tables sur PHPmyadmin et j’ai utilisé PHP pour remplir les données sur le back-office depuis PHPmyadmin. J’ai effectué des requêtes SQL pour ajouter, modifier et supprimer des éléments dans la table principale.

Le but de ce back-office est d’afficher en temps réel le contenu de la base de donnée sur le site. Le bouton Ajouter permet d’ajouter un produit selon sa catégorie sur la page précisée. Le bouton Éditer permet de modifier un élément de la base de donnée, son nom, son prix ou son image.

Projet Github search app

Projet réalisé dans le cadre de ma formation. Le but était de créer un site et d’intégrer l’API de Github à celui-ci.

Projet Github search app
  • Figma
  • HTML
  • CSS
  • JavaScript
  • Git
  • Github

J’ai développé une application web permettant de rechercher des profils existants sur le site de Github en utilisant leur API.

Je me suis basé sur la maquette qui nous a été donné sur Figma pour mettre en place la partie HTML et CSS. Sur ce projet, j’ai utilisé du SASS pour regrouper toutes les informations données sur la charte graphique.

La partie Javascript a été utilisée pour mettre en place l’API et le thème sombre. Pour le thème sombre, j’ai fait une fonction qui permet d’alterner d’un thème clair à un thème sombre au click d’une div. Pour l’API, j’ai utilisé une requête AJAX avec fetch et ai récupéré les données en JSON pour chercher les informations demandées sur la maquette. Il n’y a aucune partie Back-end sur ce projet.

Le site se compose d’un titre, d’un bouton qui permet de passer du thème clair au thème sombre et inversement, une section « recherche » où l’on peut chercher des utilisateurs Github et soit appuyer sur le bouton « search », soit appuyer sur la touche entrer pour lancer la recherche. Pour ce qui est du résultat, il se trouve dans une autre section. Il indique la photo de profil de la personne, son pseudo, son username, sa date d’inscription, sa bio, le nombre de repos qu’il a créé, le nombre de personnes qui le suivent, le nombre de personnes qu’il suit, son secteur d’activité, un lien vers son twitter, l’url de son site web et le nom de son entreprise. Tout a été mit en place à l’aide de Display : grid ; sur le CSS. J’ai également adapté la partie responsive du site pour les tablettes et téléphones avec des @media dans le CSS.

Formulaire de contact

Projet que j’ai créé dans le but d’apprendre à réaliser un envoie de mail via un form avec l’AJAX.

Formulaire de contact
  • Figma
  • HTML
  • CSS
  • Bootstrap
  • JavaScript
  • PHP
  • MySQL
  • Git
  • Github

J’ai réalisé ce projet à l’aide d’une maquette que j’ai moi même créée. Le but de ce projet était avant tout de commencer à toucher à la partie Back-end, mais également de valider des étapes Front-end.

J’ai donc commencé par faire un benchmark de plusieurs formulaires, m’en suis inspiré et ai fait des zonings, des wireframes, une charte graphique et des maquettes sur Figma. Une fois la maquette terminée, j’ai commencé l’intégration du site web en structurant le fichier HTML, puis en implémentant le CSS tout en suivant ma charte graphique.

Le formulaire contient plusieurs champs, tous fonctionnels. J’ai créé un algorithme permettant d’effectuer un envoie de mail via Javascript et PHP à l’aide du JSON et de l’AJAX.

Le site comprend un titre principal, un bouton qui permet de passer du thème clair au thème sombre et inversement ainsi que de 5 champs (tous obligatoires), un input prénom, un input nom, un input e-mail, un input sujet et un input message. Les mesures de sécurité obligent l’utilisateur à remplir les champs demandés, sinon une erreur s’affichera à l’écran indiquant les champs incomplets en rouge ainsi qu’un message demandant de bien remplir le formulaire. Si tous les champs sont bien remplis, un message de confirmation apparaît disant que la demande a bien été envoyée et les champs se réinitialisent.
